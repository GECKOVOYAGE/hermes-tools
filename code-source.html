Flux d’exécution simplifié

  1. Entrée utilisateur via Telegram, CLI ou WebUI.
  2. Construction du contexte: mémoire, skills, profil, outils actifs.
  3. Appel LLM avec schémas de tools.
  4. Dispatch tool → résultat réinjecté dans la conversation.
  5. Réponse finale + livraison sur la plateforme.

Ce qui a grossi depuis la V1

Hermes est passé d’un agent Python orienté CLI/gateway à une plateforme complète: WebUI/desktop, plugins, MCP, profils, cron robuste, TUI, registry de skills, tests massifs et packaging Docker/Nix/Windows.
